## 3. What was done (all completed and verified)

### 3.1 Blank-page bug on "Pre-Calculus - Christian 2" — FIXED
- **Root cause:** the sheet has a large passage block ("Identity Reference", a full formula sheet) at position 0. `PassageBlock` in `src/components/sheets/SheetDocument.tsx` applied `break-inside-avoid` unconditionally. A nearly-page-tall block with `break-inside: avoid` gets pushed *whole* to the next page by the browser's print engine, leaving the first page with only the cover title — the "entire blank page".
- **Fix (all cases):**
  - `SheetDocument.tsx` → `PassageBlock`: `break-inside-avoid` now applies only to short passages (≤600 chars). Long ones flow across pages, with `box-decoration-break: clone` so the bordered box splits cleanly per page.
  - Passage block **titles** (stored in DB, e.g. "Identity Reference") are now rendered in print — they were silently dropped before.
  - `globals.css` print media: KaTeX display equations never get sliced in half by a page break.
  - Same guard applied to `QuestionRenderer` for questions carrying long passages/images (new feature below).
- **Verified:** rendered the real sheet headlessly (Edge → PDF). Before: 8 pages, page 1 nearly empty (63 text ops). After: 7 pages, page 1 full (306 text ops), no near-blank page.

### 3.2 "Lines per question doesn't change spacing" — FIXED (real cause was silent save failure)
- **Root cause:** the questions on that sheet came from the **public question bank** (`owner_id = null`). `updateQuestionAction` filtered `.eq("owner_id", user.id)`, so the UPDATE matched **0 rows and reported success** — the editor showed "Saved" and the new line count, but the DB never changed, so print never changed. (The hidden lines themselves do reserve space — 17px per line — the edit was just never persisted.)
- **Fix:** copy-on-edit in `src/lib/actions/questions.ts` → `updateQuestionAction`. Editing a bank question you don't own now saves the edit as a **private copy** (keeping subject/topic/difficulty/tags/solution) and repoints this sheet's `sheet_questions` link to the copy. The client (`QuestionEditorShell` → `QuestionList`) updates its local `questionId` via a new `onQuestionIdChange` callback.
- Save failures are no longer swallowed: the editor now shows "Couldn't save — check your connection and try again" in red instead of a false "Saved".

### 3.3 Passages and/or images on questions (personal bank + everywhere) — ADDED
- **DB:** new migration `0017_question_passage_images.sql` (see §2). The `question-images` storage bucket + upload action already existed and are reused.

- **Personal bank creation/editing** (`AddToBankModal`): new "Reading passage (optional)" textarea and an "Images (optional)" upload row with thumbnails + remove. i18n'd (en + pt).
- **Sheet editor:** new `QuestionExtrasEditor` (rendered under every question type via `QuestionTypeEditor`) — "Add passage" / "Add image" for any question directly on a sheet.

## 6. Round 2 (2026-07-02, later the same day) — user-reported issues after testing

### 6.1 Root cause of "passage/lines don't stick": a save race in copy-on-edit — FIXED
DB forensics on question 3 showed the copy-on-edit flow (from §3.2) had a race: `updateQuestionAction` was keyed by `question_id`, but copy-on-edit *changes* the question id mid-editing. A debounced save issued before the client learned the new id would (a) create **another** orphan copy and (b) repoint by matching the old `question_id` — which matched 0 rows. Later edits (the passage, line counts) landed on orphans nobody references. Found and deleted one such orphan (`31ac43f8…`) from the DB after verifying it was unreferenced.

Fix, in two layers:
- `updateQuestionAction(sheetId, sheetQuestionId, content)` is now keyed by the **stable link id**; the server resolves the current `question_id` from the link at save time, and copy-on-edit repoints by the link's primary key. A stale client id can no longer strand an edit.
- `QuestionEditorShell` serializes saves: one request in flight, latest content queued behind it.

### 6.2 Editor lag — FIXED (two causes)
- Every debounced autosave called `revalidatePath("/sheets/[id]")`, which re-renders the whole route server-side after each keystroke pause. Removed from all editor-scoped autosave/add/remove/reorder actions (`questions.ts`, `groups.ts`, `sheets.ts`). Local state is the editor's source of truth; dynamic routes re-fetch on navigation anyway.
- Every keystroke re-typeset the entire live preview (KaTeX for all questions + passages). Now: `QuestionRenderer` and `PassageBlock` are `React.memo`'d, the preview inputs go through `useDeferredValue` in `SheetEditor`, and the in-editor Latex previews (statement, passage editors) are deferred as well.

### 6.3 Line counts and the lines toggle — FIXED
- New `NumberField` (`src/components/ui/Input.tsx`): buffered numeric input you can clear and retype; commits valid values as typed, clamps on blur. Applied to: per-question answer lines (open + essay), page-settings default lines, page margins, cover geometry (X/Y/W/H), bank-modal lines input.
- The "Answer lines (default)" input is no longer hidden when lines are toggled off (the space still applies); toggle relabeled "Show answer lines" with a hint that hidden lines keep their space. Essay questions now respect the toggle too (space preserved, lines invisible — same as short-answer).
- Note: hidden lines always DID reserve identical space per line; the earlier reports were the §6.1 save race making edits vanish.

### 6.4 Photos on sheet creation — ADDED
New "Photo / image" cover block type (`image` in `CoverBlockType`): upload via the existing logos bucket, drag/resize on the cover canvas, renders `object-contain` in print and preview. (Per-question images from §3.3 already covered images inside the question list.)

### 6.5 The two console errors — FIXED
- **Duplicate key warning:** `MatchingEditor` in `AddToBankModal` used a keyless `<>` fragment inside `.map()` — now `<Fragment key={lItem.key}>`.
- **Hydration mismatch (`nonce=""` vs real value):** browsers censor the `nonce` content attribute after parsing, so client hydration always reads an empty string. Added `suppressHydrationWarning` to the theme init `<script>` in `layout.tsx` (the standard remedy; CSP still works because the property survives).
